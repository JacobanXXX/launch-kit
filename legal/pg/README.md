# PG (결제대행사) 약관 추가본

> 한국 SaaS 결제 시 필요한 PG별 약관·개인정보처리방침·환불 정책 추가 항목.
> 기존 `legal/privacy.html`, `terms.html`, `refund.html`에 **추가**로 들어가는 내용입니다.

## PG 선택 가이드

| PG | 추천 대상 | 가입 난이도 | 수수료 |
|---|---|---|---|
| **토스페이먼츠** | 1인 인디 메이커, 모던 UI 선호 | ★ (5분 가입) | 2.8-3.0% |
| **포트원** | 멀티 PG 라우팅 원함 (카카오·네이버·토스 한 번에) | ★★ (1일) | 2.7-3.5% |
| **이니시스** | 보수적 B2B·교육·관공서 | ★★★★ (수일~) | 2.5-3.0% (협상 가능) |

## 어떤 약관 추가본을 써야 하나?

본인이 사용하는 PG 1개의 `.md` 파일을 골라서:
1. 기존 `legal/privacy.html`의 제5조 (위탁) 표에 PG 행 추가
2. 기존 `legal/terms.html`의 제6조 (결제) 항목에 결제 조항 추가
3. 기존 `legal/refund.html`의 제6조 (처리 기간) 표에 PG 환불 기간 추가

## 가능한 모든 PG 사용 시

여러 PG를 동시에 사용한다면 (예: 카드는 토스, 가상계좌는 이니시스):
- 각 PG의 약관 추가본을 모두 적용
- 결제수단별로 어떤 PG로 라우팅되는지 명시 권장

## 파일

- [tosspayments.md](tosspayments.md) — 토스페이먼츠
- [portone.md](portone.md) — 포트원 (구 아임포트)
- [inicis.md](inicis.md) — KG 이니시스

각 파일에는 약관 텍스트 + 결제 코드 샘플 링크가 포함되어 있습니다.

## ⚠️ 공통 면책

PG 약관은 PG사 정책 변경 시 함께 업데이트해야 합니다.
- PG사 메일 공지 구독 권장
- 본격 매출 발생 전 변호사 1회 검토 강력 권장
- 정기결제(빌링) 도입 시 별도 약관 추가 필요
